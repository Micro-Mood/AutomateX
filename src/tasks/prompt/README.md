# 提示词目录

本目录存放 AutomateX 任务引擎使用的提示词模板。引擎在每次任务执行时动态加载模板，修改后无需重启服务。

## 文件说明

| 文件 | 用途 |
|------|------|
| `select.md` | 主系统提示词，定义 AI 身份、响应方式和工具调用规则 |

## 模板变量

提示词模板支持以下变量替换（引擎自动注入）：

| 变量 | 说明 |
|------|------|
| `{cwd}` | 当前工作目录路径 |
| `{tool_list}` | 可用工具列表（紧凑格式） |

## AI 响应方式

`select.md` 中定义了 AI 的三种响应方式：

1. **直接回复** — 对话、问答等不需要工具的场景
2. **JSON 工具调用** — 返回 `{"call": [...]}` 格式的工具请求
3. **控制指令** — `done`（完成）/ `fail`（失败）/ `ask`（询问用户）/ `update_todo` / `get_context`

## 自定义指南

编辑 `select.md` 可自定义 AI 的行为，常见定制：

- 修改 AI 身份和语气
- 调整工具选择的优先级策略
- 添加特定领域的处理规则
- 定制 Windows 命令执行流程

## V3 两阶段工具调用说明

1. **SELECT 阶段**: AI 分析任务，仅返回需要的工具名称列表（不传详细参数说明）
2. **PARAMS 阶段**: 引擎注入被选工具的参数说明，AI 填写参数并执行调用

这种架构大幅减少了 Token 消耗（约 90%），因为工具详细说明只在需要时注入。
